# Security Policy

## Supported versions

The `main` branch is the supported development line for this open-source project.

## Reporting a vulnerability

Please do not open a public issue for suspected vulnerabilities.

Report security concerns privately to the repository owner or project maintainer. Include:

- A clear description of the issue.
- Steps to reproduce.
- Affected files, endpoints, or workflows.
- Potential impact.
- Suggested remediation, if known.

## Security scope

Important areas include:

- API-key validation and rotation.
- Prompt and response preview storage.
- Organisation and tenant isolation.
- Human review workflows.
- Audit log integrity.
- Database access controls.
- CI/CD and deployment secrets.

## Responsible disclosure

We aim to acknowledge valid reports, investigate promptly, and publish remediation notes once fixes are available.
